# Contribuindo com ExerciciosJava

Obrigado por considerar contribuir! 🎉

## Como contribuir

1. Faça um **fork** do repositório.
2. Crie uma **branch** descritiva: `git checkout -b feat/minha-feature`.
3. Faça suas alterações seguindo o estilo do projeto.
4. Execute `javac` para garantir que o código compila.
5. Commit com mensagem clara: `git commit -m "feat: descrição concisa"`.
6. Faça push: `git push origin feat/minha-featura`.
7. Abra um **Pull Request**.

## Convenções de commit

Usamos [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` nova funcionalidade
- `fix:` correção de bug
- `docs:` documentação
- `refactor:` refatoração
- `style:` formatação (sem mudança de lógica)

## Estilo de código

- Mantenha a convenção de nomenclatura existente nos arquivos.
- Não adicione comentários em português ou inglês a menos que seja realmente necessário.
- Prefira nomes descritivos para variáveis e métodos.
